# i18n rules

UI text uses i18next (locales `es`/`en`/`cn` under `src/config/i18n/locales`). Full guide:
`.claude/skills/i18n`.

- **Keep all three locales in sync.** `en.ts` defines the `Translation` shape; `es.ts` and
  `cn.ts` must have the EXACT same keys — `bun run type-check` fails otherwise. Add a key to
  all three at once. Chinese needs a native-speaker review before prod.
- **Never translate DATA values** (enum/model values, option `value`, comparisons like
  `status === 'Activo'`) — translate only the visible `label`/text. Translating a data value
  breaks filtering/logic.
- Use `t()` from `useTranslation()`. For non-component factories (columns/fields), pass `t`
  as a parameter and `useMemo(() => factory(t), [t])`. Reuse `common.*` before adding
  feature-specific keys; interpolate with `{{name}}`.
- Don't call `i18n.changeLanguage` manually — it auto-syncs to the shell's
  `configuration.language`.
